What is GitHub?

GitHub is a service that provides a source code repository by using Git. GitHub can be defined as a tool for supporting source code management which is arranged as distributed data. Users could use GitHub to synchronize the source code of their team to a server. It also helps developers to check the source code during the process. 

Version

GitHub has two versions: Paid and free version. The free version is usually used for saving the source code by the developers. For enterprises, they prefer the paid version because it increases team management ability and high security.

Benefits

Firstly, developers can manage the source code easier with GitHub. For instance, all the repositories of the codes will be stored on GitHub after creating. Developers can check what they or others have committed so everyone will comment on it to give their suggestions or ideas. Secondly, you do not have to worry about the project modification, GitHub always saves a different version from the original one after adjusting. As a result, you can track your project and know the person, time, content that they had edited. Thirdly, you can format your text on the web through Markdown. Markdown helps developers to edit the display of a document, such as bold or italic. In most cases, it looks like plain text with special characters such as # or * that have been added. Finally, source code is a very important factor that helps you to prove yourself when applying for a job. If you have a GitHub in the CV, the rate that you can be accepted to the company would higher than anyone. Many large companies have put GitHub into the requirement for recruitment of personnel so that they can know your strengths as well as coding skills.

Basic GitHub commands

Command Definition
git config This command is used for setting the username of the commits and it only affects the future commits without changing the names used for the previous one
git add With git add, you can add all the files in the current folder except files begin with a single dot. However, git add . will add all the files include files that have names beginning with a single dot
git commit -m You use this when you commit changes for the repository, you can write a comment in a to know what you have changed
git push You use this command to push changes to a remote repository, you must remember exactly the branch so that you will not push to the wrong one

How to resolve conflicts in GitHub

Conflicts are the situation when you and someone in your teampush something to the branch with a same name. As a result, an error will happen. To figure out it, there are some steps for you.

Read More

Step 1: Click on the Pull requests icon under your repository name.

Step 2: Look at the "Pull Requests" list, find the pull request with a merge conflict that you would like to resolve, and click on that. 

Step 3: Click Resolve conflicts, you can find it near the bottom of your pull request.

If you have an error, look for more information here: "Resolving a merge conflict using the command line."

Step 4: Consider carefully if you want to keep only your branch's changes or the other branch's changes. In the case of making a brand new change, it may combines changes from both branches. Before making the changes in the final merge, delete the conflict markers <<<<<<<, =======, >>>>>>>. 

Step 5: With more than one merge conflict in your file, scroll down to the next set of conflict markers and repeat steps four and five for resolving the merge conflict. 

Step 6: When finish resolving all the conflicts, click Mark as resolved icon.

Step 7: If you have more than one file with a conflict, Only selecting the next file you want to edit on the left side of the page under "conflicting files" when there is more than one file with a conflict, after that, repeat steps from four to seven. When all of your pull request's merge conflicts are resolved, it means that you have finished.

Step 8: Click Commit merge so it will merge the entire base branch into your head branch. 

Step 9: If there is a notification, review the branch that you are committing to. 

You can choose to update the head branch with the changes you had made to resolve the conflict before if this is the default branch of the repository. You can also create a new one and use this as the head branch of the pull request. 
Enter a name if you decide to create a new branch. 
You must create a new branch if the head branch of your pull request has been protected. You will not get the option to update it. 
Click Create branch and update my pull request or I understand, continue updating BRANCH.

Step 10: Click Merge pull request to merge your pull request.

Branching

A branch is mean that you separate your work without affecting others in the repository. In a repository, there is always a default branch called main or master. However, you can add many branches later. You can pull request to merge it together so that you can have a complete project
